简单工厂 ,工程类里if ... else 指定name即可
if("apple".equals(name)){
	return new Apple();
}else if("orange".equals(name)){
	return new Orange();
}else{
	return new Bananer();
}

缺点是如果要增加Peach,要改代码，由次引出了工厂方法

工厂方法，生产水果的方法抽象成一个抽象方法。
AppleFactory(){
	public Fruit getFruit(){
		return new Apple();
	}
}

OrangeFactory(){
	public Fruit getFruit(){
		return new Orange();
	}
}

BananerFactory(){
	public Fruit getFruit(){
		return Bananer();
	}
}

缺点是
1.要增加很多类
2.代码里必须知道是哪个工厂,但绕一圈又回到了起点去了，又要写一堆的if else

由2又引出了抽象工厂，就是用一个abastractFactory
